# Роли Pocket MVP

Документация описывает пользовательские роли, доступные экраны и ожидаемое поведение веб-приложения. Текущий frontend работает на демонстрационных данных; разделы `Интеграция` фиксируют контракт, который потребуется подключить к Go backend.

| Роль | Назначение | Документ |
| --- | --- | --- |
| Владелец заведения | Управление заведениями, меню, командой, заказами и финансами | [venue-owner.md](./venue-owner.md) |
| Гость заведения | Поиск заведений, заказ, оплата и бронирование | [venue-customer.md](./venue-customer.md) |
| Сотрудник | Обслуживание столов и обработка заказов во время смены | [venue-staff.md](./venue-staff.md) |
| Пользователь приложения | Регистрация, вход, аккаунт и выбор доступной роли | [application-user.md](./application-user.md) |

## Общие правила доступа

- Роль определяет доступную навигацию и набор операций.
- Владелец может переключать активное заведение; данные экранов должны загружаться в контексте выбранного `venueId`.
- Гость работает с каталогом заведений и не закреплен за одним заведением до открытия меню, брони или заказа.
- Сотрудник видит только назначенное заведение, активную смену и разрешенные менеджером операции.
- Backend должен проверять права независимо от того, какие элементы скрыты во frontend.
