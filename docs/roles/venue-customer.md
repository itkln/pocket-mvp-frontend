# Гость заведения

## Назначение

Гость ищет заведения, открывает публичное меню, бронирует стол, формирует заказ и оплачивает счет. До выбора заведения пользователь находится в общем каталоге Pocket и не привязан к конкретному месту.

## Доступный функционал

| Экран | Возможности |
| --- | --- |
| Заведения | Поиск, быстрые фильтры, карточки заведений и сохраненные места |
| Меню | Категории, поиск, карточки блюд, избранное и добавление в заказ |
| Корзина | Количество позиций, комментарии, чаевые, способ оплаты и итоговая сумма |
| Бронь | Дата, время, количество гостей, конкретный стол, комментарий и предзаказ |
| Мои заказы | Активный заказ, прогресс приготовления, история, повторный заказ, отзыв и отправка ссылки друзьям |
| Профиль | Контакты, сохраненная карта, предпочтения и выход |

## Ключевые сценарии

### Заказ по QR-коду

1. Гость сканирует код физического стола.
2. Backend проверяет токен и возвращает `venueId` и `tableId`.
3. Гость выбирает блюда и подтверждает заказ.
4. Заказ и счет автоматически связываются со столом.
5. Статус заказа обновляется в реальном времени.

### Бронирование и предзаказ

1. Гость выбирает дату, время и количество гостей.
2. Приложение показывает доступные столы.
3. Гость выбирает конкретный стол и при необходимости добавляет блюда.
4. Backend временно удерживает слот и подтверждает бронь после финальной отправки.

### Оплата

1. Гость проверяет заказ и выбирает чаевые.
2. Выбирает сохраненную карту или wallet payment.
3. Payment provider подтверждает операцию.
4. Backend закрывает счет только после успешного webhook.

## Права

- Просмотр опубликованных заведений и доступных позиций меню.
- Создание собственных броней, заказов, платежей и отзывов.
- Просмотр только своей истории и активных заказов.
- Изменение только собственного профиля и платежных методов.

## Интеграция

- Публичный каталог и меню должны поддерживать cache и пагинацию.
- Корзина хранится локально до создания server-side order draft.
- Брони требуют защиты от двойного резервирования.
- История заказов и отзывы доступны только после авторизации.
- Передача заказа друзьям использует безопасную read-only ссылку с ограниченным сроком действия.
