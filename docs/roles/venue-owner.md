# Владелец заведения

## Назначение

Владелец управляет одним или несколькими заведениями из общего рабочего пространства. Активное заведение выбирается в верхней части боковой панели, после чего все операционные экраны используют его `venueId`.

## Доступный функционал

| Экран | Возможности |
| --- | --- |
| Обзор | Выручка, количество заказов, средний чек, загрузка зала, каналы продаж и текущие заказы |
| Заказы | Просмотр заказов из зала, сайта и самовывоза; фильтрация по статусу; открытие деталей заказа |
| Меню | Категории, поиск позиций, сеточный и списочный вид, добавление и редактирование блюда, доступность, дублирование, перенос, скрытие и удаление |
| План зала | Столы, места, состояние, официант, текущий счет, открытие счета и QR-коды |
| Команда | Приглашение сотрудников, назначение ролей, просмотр статуса смены и удаление доступа |
| Аналитика | Выручка, загрузка по времени, популярные позиции и агрегированный рейтинг |
| Отзывы | Поиск, фильтрация по столу и оценке, переход к заказу, публикация и редактирование ответа |
| Заведение | Публичный профиль, изображения, контакты, адрес, расписание, каналы заказов и уведомления |
| Платежи | Статус подключения, баланс, выплаты, транзакции и экспорт |
| Подписка | Тариф, период оплаты, платежная карта, реквизиты, VAT и подтверждение подписки |
| Аккаунт | Личные данные владельца, пароль, безопасность и выход |

## Ключевые сценарии

### Переключение заведения

1. Владелец открывает список заведений в боковой панели.
2. Выбирает нужное заведение.
3. Frontend сохраняет новый `venueId` и повторно загружает данные текущего экрана.
4. Breadcrumb, публичный предпросмотр меню и ссылки QR-кодов используют выбранное заведение.

### Управление меню

1. Владелец создает категорию.
2. Добавляет позицию с названием, описанием, ценой и изображением.
3. Назначает категорию и доступность.
4. Проверяет результат в desktop или mobile предпросмотре.
5. При необходимости дублирует, переносит, скрывает или удаляет позицию.

### QR-код стола

1. Владелец открывает раздел QR-кодов на плане зала.
2. Скачивает код одного стола или архив всех кодов.
3. При компрометации ссылки перегенерирует код.
4. Backend увеличивает версию токена и делает предыдущую ссылку недействительной.

## Права

- Полный CRUD для собственных заведений, меню, столов и сотрудников.
- Просмотр заказов, аналитики, отзывов и платежей только своих заведений.
- Управление подпиской и аккаунтом на уровне рабочего пространства.
- Назначение сотрудникам только разрешенных системных ролей.

## Интеграция

- `GET /owner/venues` и `POST /owner/venues` для списка и создания заведений.
- Все venue-зависимые запросы должны содержать `venueId` в path или query.
- Меню и категории требуют CRUD endpoints и загрузку изображений через object storage.
- Заказы, статусы и план зала должны обновляться через WebSocket или SSE.
- Платежи и подписка должны использовать provider checkout и webhook-подтверждение, а не доверять состоянию frontend.
